Getting Started with lwdid

Introduction

The lwdid package implements the Lee and Wooldridge (2025, 2026) simple transformation approach for Difference-in-Differences (DiD) estimation with panel data. The key innovation is a unit-level time-series transformation that converts panel DiD into a cross-sectional treatment effects problem.

Key features:

  • Two designs: Common Timing and Staggered Adoption
  • Four estimators: RA, IPW, IPWRA, PSM
  • Robust inference: HC0–HC4, cluster-robust, Wild Cluster Bootstrap, Fisher Randomization Inference
  • Exact small-sample inference: valid even with \(N_1 = 1\)
  • Diagnostics: parallel trends tests, sensitivity analysis, clustering diagnostics

Installation

# From GitHub
# install.packages("devtools")
devtools::install_github("gorgeousfish/lwdid-r")

Data Format

lwdid requires long-format panel data where each row is one unit-period observation. Required columns:

Role Parameter Type
Unit ID ivar integer or character
Time tvar integer
Outcome y numeric
Treatment (Common Timing) d + post binary (0/1)
Treatment (Staggered) gvar integer or NA

Demean vs Detrend

The rolling parameter controls pre-treatment transformation:

  • "demean" (default): Subtracts the pre-treatment mean. Requires parallel trends. At least 1 pre-treatment period.
  • "detrend": Removes unit-specific linear trends. Allows heterogeneous trends. At least 2 pre-treatment periods.

Common Timing: California Proposition 99

The smoking dataset contains the classic California tobacco control study (1970–2000). California is the sole treated state (\(N_1 = 1\)), with treatment starting in 1989.

library(lwdid)
data(smoking)

result <- lwdid(
  data = smoking,
  y = "lcigsale",
  ivar = "state",
  tvar = "year",
  d = "d",
  post = "post",
  rolling = "demean",
  estimator = "ra"
)

print(result)

Staggered Adoption: Castle Doctrine Laws

The castle dataset examines the effect of Castle Doctrine laws on homicide rates across US states. States adopted the law at different times between 2005 and 2009.

data(castle)

result_stag <- lwdid(
  data = castle,
  y = "lhomicide",
  ivar = "sid",
  tvar = "year",
  gvar = "gvar",
  rolling = "demean",
  estimator = "ra",
  aggregate = "cohort"

For a single aggregate = "overall" summary, lwdid weights cohort-time effects by treated-unit cohort sizes. Exports keep those cohort_weights; effective_weights report the post-estimation sample weights if observations are dropped from the regression sample.

Event Study

Event-study plots use relative event time. By default, the plot adds a visual anchor at event time -1 when pre-treatment periods are displayed and no observed -1 row is already in the plotted data. If you hide pre-treatment periods, the anchor is also omitted. A ref_period normalizes estimates to an observed event time, must be present in the plotted data, and does not relabel that observed row as a visual anchor.

result_event <- lwdid(
  data = castle,
  y = "lhomicide",
  ivar = "sid",
  tvar = "year",
  gvar = "gvar",
  rolling = "demean",
  estimator = "ra",
  aggregate = "event_time"
)

plot(result_event)

For aggregate = "event_time", the printed scalar ATT summarizes valid cohort-period (g, r) effects. The event-study table and plot report WATT(e) rows by relative time, with cohort-size weights renormalized over the cohorts available at each event time. WATT(e) standard errors use the stored cohort-specific standard errors with diagonal aggregation; cross-cohort covariance is not modeled unless a future joint covariance representation is added.

Use extract_effects(result_event, type = "event_time_contributions") to inspect the cohort weights and cohort-specific contributions behind each WATT(e) row. The same contribution table can be written with to_csv(result_event, file = "event-time-contributions.csv", what = "event_time_contributions") or read from to_dict(result_event)$event_time_contributions. Use to_latex(result_event, include_event_time = TRUE) to export the WATT(e) rows with standard errors, confidence intervals, contributing-cohort counts, and compact support/overlap metadata. Plot data returned by plot_event_study(result_event, return_data = TRUE) carries the same standard-error metadata for the plotted event-time rows, including se_aggregation and covariance_assumption. For non-RA event-time results, the returned plot data also preserves available overlap and support-count summaries, such as propensity-score ranges, weight dispersion, and treated/control counts.

# Post-treatment path only
plot(result_event, show_pre_treatment = FALSE)


# Normalize to the treatment period without relabeling it as an anchor
plot(result_event, ref_period = 0)

Control Variables

Controls improve efficiency and relax the parallel trends assumption. They must be time-invariant (or pre-treatment averages):

result_ctrl <- lwdid(
  data = castle,
  y = "lhomicide",
  ivar = "sid",
  tvar = "year",
  gvar = "gvar",
  rolling = "demean",
  estimator = "ra",
  controls = c("police", "income", "poverty", "unemployrt"),
  aggregate = "cohort"
)

Robust Variance Estimation

vce Description
NULL OLS (homoskedastic)
"robust" / "hc1" HC1 heteroskedasticity-robust
"hc3" HC3 (recommended for small samples)
"cluster" Cluster-robust (requires cluster_var) 
# HC1 is recommended for the smoking data (single treated unit makes HC3
# numerically unstable due to hat value = 1)

  y = "lcigsale",

  tvar = "year",

  rolling = "demean",
  estimator = "ra",

Parallel Computation

For staggered designs with many cohort-period pairs, enable parallel processing:

result_par <- lwdid(
  data = castle,
  y = "lhomicide",
  ivar = "sid",
  tvar = "year",
  gvar = "gvar",
  rolling = "demean",
  estimator = "ra",
  aggregate = "cohort",
  parallel = TRUE,
  n_cores = 4
)
